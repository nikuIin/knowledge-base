[[Удаленная работа с репозиторием git]]

**!** Всегда перед `push` нужно делать `git fetch` ([[git fetch. Подгружаем обновления с удаленного репозитория безопасно]]) **!**

Команда для того, чтобы отправить изменения в репозиторий:

```bash
git push
```

Когда мы клонируем или делаем ремоут, то у нас появляется `remote origin`. Команда `git push` в своем изначальном плане пушит изменения в дефолтной ветке (`master`/`main`) в удалённый репозиторий.

Чтобы запушить другую ветку, нужно ее закинуть в репозиторий. Это делается двумя способами:
* с флагом `-u` или без него

Флаг `-u` апстримит ветку локальную с веткой в удалённом репозитории. Это означает, что при нахождении на ветки, которую заапстримили и при пуше его, можно использовать команду `git push`, `git` все поймет. 

Иначе же нужно указывать куда мы пушим (`origin`) и какую ветку. 

```bash
git push origin my-cool-branch 
```
