[[Версионность git (коммиты)]]

`git log` считывает в своем дефолтном состоянии последний сделанный вами коммит и идет по родительским коммитам, пока не найдет того, у которого нету предка. 

Понятное дело, опять же, в состоянии по умолчанию `git log` выводит информацию с текущего (куда указывает `HEAD` (указатель позиции, выбранный коммит)) коммита историю, => он не покажет вам коммиты веток, которые никак не пересекаются с текущей. 

Но `git` знает о этих ветках и с помощью некоторых флагов мы можем кастомизировать вывод логов.

* `--oneline` — краткий и полезный вывод
* `--graph` — выводим в виде графа
* `--abbrev-commit` — сокращаем `id` коммита
* `--all` — выводим **все** ветки и **коммиты**

*Напоминание: `git` показывает текущую позицию как `HEAD`*

В итоге получается такая команда:
```bash
git log --oneline --abbrev-commit --all --graph
```

Более красивый вывод у меня стоит алиасом (`git lg`):
```bash
git log --color --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --branches
```
