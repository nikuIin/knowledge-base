`Record` в TypeScript — это *аналог словаря* в Python, но с рядом важных отличий:

- `Records` — **строго типизированные**
- В отличие от словаря в Python:
  - **ключи имеют один определённый тип**
  - **значения имеют один определённый тип**

То есть, мы **не можем использовать разные типы данных для ключей и разные типы данных для значений** одновременно.

Пример для понимания:

- В Python словарь может содержать ключи и значения любых типов:
  ```python
  my_dict = {1: "one", "two": 2}
  ```
  
- В TypeScript с `Record` тип ключей и значений фиксирован и строго контролируется:
  ```typescript
  type MyRecord = Record<string, number>;
  const record: MyRecord = {
    "a": 1,
    "b": 2
  };
  ```

Таким образом, `Record` помогает обеспечить **безопасность типов** и предотвращает ошибки, связанные с неправильным использованием ключей и значений.
