[[nginx настраиваем простое обратное проксирование на статические файлы (html, css, etc)]]

Как мы знаем, все данные в компьютере — это наборы байт. У каждого файла, то `hmtl`, `css`, eg. есть свой тип при передаче через интернет. Он определяется в заголовке `Content-type`. Для `html` это `text/html`, для `css` это `text/css`, для `png` `image/png`, etc. 

Так вот, `nginx` должен обязательно проставлять эти заголовки, иначе браузеры могут неправильно считать фалы. И своём **дефолтном** поведении `nginx` ставит `Content-type` = `text/plain`, и когда мы, например в проект внедряем `css` файл, то он будет неверно отправлен браузеру. 

Поэтому нужно определить новый `контекст` в конфиге `nginx`, а именно `types` и перечислить там все нужные нам типы:

```nginx
http {

  types {
    text/css  css;
    text/html html;
  }

  server {
    listen  8080;
    root  /Users/ivanaleksandrovci/code/test_static/;
  }
}

events {}
```

Но это **не удобно**, мы рискуем **забыть** указать тип файла здесь. Да и что, никто до этого на это уже не тратил свое время? Хороший вопрос и верный. Когда мы устанавливаем `nginx`, вместе с ним устанавливаются `mime` типы — стандарт определения файлов.

Лежит этот файлик (по крайней мере у меня) в директории:

```bash
/opt/homebrew/etc/nginx/mime.types
```

Давайте воспользуемся этим файлом:

```nginx
http {

  include mime.types;
  server {
    listen  8080;
    root  /Users/ivanaleksandrovci/code/test_static/;
  }
}

events {}

```

**Если поломали mime.types**: разработчики `nginx` о нас, дебилах, позаботились, поэтому добавили и `mime.types.default`. Впрочем, как и остальные `default` файлы.
