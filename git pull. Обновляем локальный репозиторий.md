**НИКОГДА НЕ ИСПОЛЬЗУЙТЕ git pull**

`git pull` = `git fetch` + `git merge`

Эта команда запулит изменения из удалённого репозитория в наш локальный клон. Важно понимать, что она изменит указатель `HEAD` на ее последний коммит. 

Можно воспользоваться более удачными применениями команды `git pull`:
Если мы сделали в локальном проекте новый коммит и хотим подгрузить новые коммиты с удаленного репозитория (то есть у нас ситуация, что мы обновили старую версию `main` ветки, сделали там коммит, но его еще не запушили, но другие разработчики сделали и запушили свои изменения в `main`)
Мы должны скачать их и создать коммит слияния. 
Что последний актуальный коммит в удаленном репозитории слился с нашим последним коммитом на локале.

```bash
git pull -ff
```


Второй вариант, провести `rebase`. В этом случае наш локальный коммит встанет после всех коммитов в удаленном репозитории:

```bash
git pull --rebase
```

Как мы уже сказали, никогда лучше не использовать команду `git pull`. Но если все же вы решили ее использовать, используйте эту вариацию:

```bash
git pull --ff-only
```

Эта команда просто разветвит новые изменения с удаленного репозитория с вашим локальным, а дальше вы сможете сделать либо коммит слияния, либо `rebase`.

Хорошее описание всех команд есть в вышепредставленном видосе.
