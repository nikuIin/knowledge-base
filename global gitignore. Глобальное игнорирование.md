Не знаю, хороший это тон или нет. Сидишь ты, пишешь проект, не понимаешь почему игнорируется файл `lala.txt`, в локальном `.gitingore` его нету и все. Что делать? Не понимаешь. 

А на самом деле он игнорируется глобально. Наверное это одно из свойств, о котором стоит знать, чтобы не использовать.

1. Создаем файл `~/.gitignore-global`
2. Прописываем туда все, что нужно всегда игнорировать
3. Заносим в конфиг ссылку на этот файл:

```bash
git config --global core.excludesfile ~/.gitignore-global
```

Хотя, я тут подумал, можно как раз занести все файлы для работы нашей системы. Например тот же `.DS_STORE` для мака.

Можно еще проверить почему файл игнорируется (каким `gitignore`):

```bash
git check-ignore -vv path-to-file
```
