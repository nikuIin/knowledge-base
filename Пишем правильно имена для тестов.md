Имя для теста — лучшее его `summary`.

Не стоит бояться длинных название, стоит бояться того, что имя теста будет бесполезно.

Совет по написанию название для тестов:
- пиши их так, чтобы человек, не являющийся программистом, понял, что происходит
- не используйте словосочетания `should be`, тесты это про четкость, либо да, либо нет. Меняйте `should be` на `is`
- не используйте **имена методов** в тестах **!!!!** Юнит — это про единицу поведения, а не метода/класса. `купить_вино` тест может состоять из вызова 5 методов.
- передавайте результат теста в его названии

Пример хороших имен для тестов:
```python

def test_discount_for_premium_user_is_higher_then_20():
	...
	
def test_cant_order_already_booked_wine():
	...
```

Еще раз, имена тестов <b><u>описывают поведение</u></b>, а не фукнцию, которая принимает участие в тестах.

Если же тест у нас параметризированный ([[pytest параметризованное тестирование (parametrize tests)]]), то стоит также описывать поведение, просто общее:

```python

@mark.parametrize(...)
def test_detect_invalid_delivery_date():
	...
```

`test_detect_invalid_delivery_date()` метод может в себя включать тесты, касающиеся доставки в разные промежутки времени.

Параметризированные тесты стоит разграничивать на ошибочные и валидные.
