[[Версионность git (коммиты)]]

## Отмена изменений в рабочем каталоге

```bash
git restore .
```

Это команда в своем базовом случае восстановит рабочий каталог файлом из индекса. То есть вы, если внесли изменения в файл, но не закоммитили его, то можете использовать команду `restore`, она переместит копию из идекса в ваш рабочий каталог и => восстановит прежнюю версию файла. 

В параметры `restore` можно передавать сразу несколько аргументов:

```bash
git restore file_a file_b file_c
```

## Отмена изменений в индексе

Как нам восстановить из локального хранилище версию в индекс? И вообще, когда нам это понадобится? 
Ну, понадобится нам это тогда, когда мы внесли изменения и закоммитили их, а потом осознали, что нам нужно откатить файл назад. Как это сделать нам подсказывает сама команда `status`:

```bash
git restore --staged file_for_restore
```

**Самое здесь главное** – это то, что флаг `--staged` восстанавливает файл из **ОБЪЕКТНОГО ХРАНИЛИЩА** в **ИНДЕКС**, то есть он рабочий каталог НИКАК не трогает. Это важно понимать. Еще раз, эта команда заменит файл в индексе, заберет из локального хранилища копию и вставит ее в индекс. Чтобы восстановить из индекса в рабочий каталог, используем `restore` без флагов.
