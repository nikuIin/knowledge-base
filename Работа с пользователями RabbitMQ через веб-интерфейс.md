[[RabbitMQ.canvas|RabbitMQ]]

Для работы в веб интерфейсе `RabbitMQ` нужен специальный плагин — `managment plugin`, подробнее о нем можно почитать [по ссылке](https://www.rabbitmq.com/docs/management)

Чтобы попасть в `GUI`, у пользователя должен быть к нему доступ по тегу. 
Базовые теги:
-	`administrator` — полный доступ к интерфейсу управления RabbitMQ и возможность выполнять административные действия.
-	`monitoring` — доступ к просмотру статистики и метрик, не даёт права изменять настройки.
-	`management` — доступ к веб-интерфейсу управления (ограниченный по сравнению с administrator).
-	`policymaker` — возможность управлять политиками RabbitMQ.
-	(пустой тег) — пользователь без специальных ролей.

Изменение тегов через `CLI` (команда именно задаёт теги, поэтому она **перезапишет теги пользователя, если они стоят**):

```bash
./rabbitmqctl set_user_tags <user> <tag_1> <tag_2> ... 
```

Если мы хотим убрать теги у пользователя, то просто никакие теги в команде не прописываем:

```bash
./rabbitmqctl set_user_tags <user>
```
