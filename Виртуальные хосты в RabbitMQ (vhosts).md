Чтобы разграничить сервисы между собой, а также между типами (прод, тестовая среда, препрод) в `RabbitMQ` есть виртуальные хосты — `vhosts`. 

У каждого виртуального хоста есть свои очереди. Именно по имени хоста выдаются права для пользователя. Подробнее в [[Работа с пользователями RabbitMQ, права пользователей]]

#### Посмотреть список виртуальных хостов в `CLI`

```bash
./rabbitmqctl list_vhosts
```

#### Создать виртуальный хост

```bash
./rabbitmqctl add_vhost <host_name>
```

#### Посмотреть, кто имеет права на хост

```bash
./rabbitmqctl list_permissions -p <host_name>
```
