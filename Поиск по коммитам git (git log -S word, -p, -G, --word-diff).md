Зачастую существуют ситуации, когда нам нужно найти в репозитории часть, которая была изменена некоторым коммитом. Для этого подходит команда:

```bash
git log -S word
```

Она найдет все коммиты, где фигурирует изменилось количество слов "*word*". То есть где оно добавилось или удалилось. Те коммиты, где количество слов не изменилось, **не будут выведены**.

Флаг `-p` также предоставит автоматически вывод `diff` к каждому найденному коммиту.

```bash
git log -pS word
```

Также после команды можно указать еще один флаг: `--word-diff`, чтобы видеть изменения на уровне слов.

```bash
git log -pS word --word-diff
```

Смешав команды вывода с кастомизационными, можно получить удобный и информативный вывод (у меня алиас `lg`):

```bash
git lg -pS my-cool-word-for-searching --word-diff
```

# Поиск по изменениям

Если перед нами стоит задача найти все коммиты, которые оперировали (изменяли, добавляли или удаляли) определенное слово, то можно использовать флаг `-G`. Он похож на `-S`, но в отличие от него он выведет все коммиты, в которых строка с искомым значениям менялась (то есть участвовала в `diff`). Вкратце: выведется все коммиты, где в `diff` оперирируется поисковая строка.

Также `-G` поддерживает регулярные выражения.

```bash
git lg -pG "^(word|word2)$" --word-diff
```
