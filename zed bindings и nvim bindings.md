## Terminal

1. Open/close terminal `cntrl-/` или `space f t`
2. Open new terminal window `cmd-n`
3. Open central terminal and open __lazygit__ `space l g`.
   `<leader>gg` в `nvim`
## Code

1. `g r` — see all references (where this code are using)
2. `s s` — офигенная вещь, бегаем по функциям очень быстро (`<leader>ss` в `nvim`)
3. `<leader> c a` — список быстрых решений проблемы
4. ` w` — закрыть панель (например закрыть `lazygit`)
5. `z a` — скрыть кусок кода
## Multicursor

1. `g a` или `cmd + shift + l` — выбрать мультикурсором все вхождения слова
2. `g l` — выбрать следующее вхождения мультикурсором
3. `cmd + alt + [клавишы внизи/вверх/влево/вправо]` — поместить курсор на строку/позицию
## Splits 

1. `⌘ + shift + \` — сплит горизонтальный
	- `⌘ + [1,2,3,4]` — создать сплит с номером (потом можно по ним также переключаться)
2. `⌘ + k right/left/up/down` — сплиты
## Поиск по проекту

1. `S s` — офигительный поиск по тексту
2. `Space /` — `grep`
## Сдвиг строк

1. `Tab` / `Shift-Tab`
2. Выделяем, потом `shift- >` /  `shift- < ` — для одного сдвига, после сдвига выделение пропадет
3. Выделяем, потом `cmd-]` / `cmd-[` — для сдвигов без сброса выделения

## Прыгаем по коду

1. gg` — наверх
2. `G` — в конец
3. `[code_number]gg` — на строку с номером `[code_number]`, например `125gg` — переместит нас на 125 строку
4. `cntrl-o` — вернуться на место, откуда прыгнули
5. `cntrl-i`

## Использование меток (marks):

- Находясь на строке 125, нажми `ma` (это создаст метку с именем a).
- Переместись в начало файла (`gg`) или куда угодно.
- Чтобы вернуться к строке 125, нажми *\`a* (обратный апостроф + a) — это вернёт тебя точно на позицию метки.
- Или используй `'a` (одинарная кавычка + a), чтобы вернуться в начало строки с меткой.

## Ищем `char` в строке

1. `f{character}` — перенесет курсор перед `{character}`, находящихся **справа** от курсора. 
	* `;` — следуещее вхождение
	* `,` — предыдущее вхождение
2. `F{character}` — перенесет курсор перед `{character}`, находящихся **слева** от курсора. 
	* `;` — следуещее вхождение
	* `,` — предыдущее вхождение
