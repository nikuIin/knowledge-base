[[Git (конфиг, алиасы). git alias]]

Как работае гит? Как он хранит изменения? 

Есть две области: индекс (временная полка) и объектная база данных (изменения).

При `CRUD` операций на файл, операция заносится в индекс.
Делая `git add` мы помечаем файлы во временной папке, а `git commit` уже переносит из временного хранилища в объектное.  При добавлении в индекс `git` **копирует** ваши файлы, а не перемещает их. 

У коммита есть хэш — он состоит из имени и почты пользователя, а также времени самого коммита. 

Все данные в гите хранятся в двойном формате, это позволяет компьютеру еще быстрее работать с ними.
