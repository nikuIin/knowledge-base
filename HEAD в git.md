`HEAD` – это ссылка на определенную ветку (есть исключение), то есть это транзитивная ссылка на коммит ([[Ветки в git]]). 

Его главная цель — показать где мы в текущий момент находимся. 

С помощью него определяется родитель для следующего коммита, а именно текущий коммит, на который весит закрепка `HEAD`. 

Поэтому важно понимать в каком месте проекта вы находитесь, поэтому для профилактики почаще используйте этот список команд:

```bash
git log --oneline --graph --all
git status
git branch
```


## Относительность от `HEAD` (*~* и *^*)
Еще, благодаря `HEAD`, можно относительно его ссылаться на другие коммиты.

`HEAD~1` — 1 коммит вниз от `HEAD`
`HEAD^1` — первый (левый) родитель коммита `HEAD`

Смешиваем: `HEAD^2~2` — второй коммит от второго родителя. 

Также нужно понимать, что `HEAD~` = `HEAD~1`.  И `HEAD~1` в коммите слияния ([[Слиянение веток git (merge)]]) будет идти по левому (первому) родителю.

Я провожу следующую аналогию:

*^1* — вниз первый родитель
*~1* — 1 раз вниз

При сложных масках лучше просто использовать идентификаторы коммитов, а то так можно запутаться.
