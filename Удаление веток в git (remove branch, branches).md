[[Ветки в git]]

Когда стоит удалять ветку?
Когда работа завершена и эта ветка слита в интеграционную, дальше в ней нету смысла, дальше это как использованный стаканчик из под кофе, выкинь его в мусорку. 

Чтобы удалить ветку, выполни эту команду:

```bash
git branch -d my_branch
```

#создание-ветки-по-коммиту
Самое классное в том, что `git` вернет коммит, на который ветка ссылалась, а как мы помним, ветки – это просто ссылка на коммиты. А => мы можем восстановить ветку, есть такая для этого команда:

```bash
git branch my_branch 424a32b...f94f
```

При удаление ветки удаляется просто ссылка на коммит, сам же коммит остается. К слову, коммит удаляется только когда, когда на него не ссылается ни одна ветка или у него нету дочерних элементов. 

Из этого следует, что если мы удаляем ветку, которая не была никуда слита, то мы удаляем и все изменения. `git` об этом заботится и нас об этом предупреждает, поэтому нам нужно использовать принудительное удаление:

```bash
git branch -D my_branch
```
