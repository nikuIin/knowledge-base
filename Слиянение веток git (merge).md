[[Ветки в git]]

Слияние веток также является одним их самых частых кейсов при работе с `git-ом`. 

## Слияние «Ускоренная перемотка»

Чтобы слить две ветки, нам нужно переместиться на базовую и выполнить команду слияния с дополнительной (функциональной веткой):

```bash
git switch master;
git marge my_branch;
```

Что произойдет в простом случае? Если `my_branch` просто отсоединялась от `master` раньше, то при `merge`-операции указать (стикер) ветки `master` просто переместится на последний коммит ветки `my_branch`. Это называется *ускоренной перемоткой*. То есть слияние – просто перемещение указателя последнего коммита. 

И тут важно понимать, что **направление слияния имеет значение!**
Если мы сделаем:
```bash
git switch my_branch;
git merge master;
```

То выпадет сообщение *Already up to date.* Почему? Потому что `my_branch` это и есть производная ветка ветки `master`, => обновлять нечего, все что было в `master` если и в `my_branch`. 
## Коммит слияния

Коммит слияние отличается от предыдущего тем, что он сливается с другой веткой, имеющую того же родителя. 

Например, у нас была изначальная ветка `master` и ветки `b1`, `b2`. На `b1` сделали *n* изменений и смерджили с `master` (ускоренная перемотка). Теперь сделали *m* изменений в `b2` и нужно слить все тоже в `master`. Что тут произойдет? Тут уже не будет ускоренной перемотки, так как мы просто тогда потеряем изменения ветки `b1`, тут у нас создастся новый коммит, которые объединит наши ветки. `git` сам при `merge` откроет наш редактор по умолчанию и попросит ввести коммит. Или же мы можем воспользоваться такой командой:

```bash
git merge b2 -m "Merge b2 to master"
```

Уникальность этого коммита в том, что его создаёт сам `git` и что он имеет двух родителей.
