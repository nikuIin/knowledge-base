[[Ищем «плохой» коммит. Бинарный поиск. git bisect]]

Мы можем автоматизировать (если у нас есть тесты) поиск первого плохого коммита. 

Для этого нужно написать `bash-script`:

```bash
#!/bin/bash

# Выполняем поиск строки "blinnis" без учета регистра
rg -i blinnis >/dev/null

# Проверяем код выхода команды rg
if [ $? -eq 0 ]; then
    exit 1 # Совпадения найдены
else
    exit 0 # Совпадений нет
fi
```

Мы должны вернуть для коммита `0` – если он хороший, иначе `1`

Далее инициализируем `bisect`, указываем хорошие и плохие коммиты, а далее запускаем тест:

```bash
git bisect run ../test.sh
```

Я выношу тесты за репозиторий `git`, ведь иначе в коммите его (файла теста) может не оказаться.
