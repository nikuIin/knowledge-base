У каждого пользователя в `RabbitMQ` есть определенные права. Всего их может быть 3:
- `read` - читать
- `write` - писать
- `configure` - конфигурировать

Задаются права на виртуальные хосты ([[Виртуальные хосты в RabbitMQ (vhosts)]]) — разграничителями между сервисами и их типами. Формат права — регулярное варажение, если имя элемента в хосте подходит под регулярку, то пользователь будет иметь такое право на соответствующий элемент.

Добавить права:

```bash
./rabbitmqctl set_permissions -p <host> <user> <regular_for_configure> <regular_for_write> <regular_for_read> 
```

**\*** Лучше экранировать «\*», если она используется в регулярках.

Чтобы пользователь имел доступ к веб-интерфейсу, ему нужно выдать тег, подробнее в [[Работа с пользователями RabbitMQ через веб-интерфейс]]
## Команды `CLI`

1. Создание

```bash
./rabbitmqctl add_user <user_name>
```

2. Удаление
```bash
./rabbitmqctl delete_user <user_name>
```

3. Изменение пароля

```bash
./rabbitmqctl change_password <user_name>
```

4. Просмотра списка пользователей

```bash
./rabbitmqctl list_users
```

5. Список прав

```bash
./rabbitmqctl list_permissions # если добавить -p можно указать нужный хост для просмотра кто имеет права к нему
```
