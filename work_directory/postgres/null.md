
```sql
select 5 + null;
--> null
```
﻿﻿``GROUP BY`` объединяет все ``NULL``-значения в одну группу;

﻿﻿``DISTINCT`` оставляет только одно значение ``NULL``;

﻿﻿функция ``AVG`` не учитывает ``NULL``, и сумма значений поля делится на количество ненулевых значений;

﻿﻿если список, модифицированный оператором ``ALL``, содержит ``NULL``, то результирующий запрос будет пуст, т.к. нельзя сравнить никакое значение с ``NULL``;

﻿﻿выражение ``<>ANY(...)`` не выполняется для ``NULL``.
