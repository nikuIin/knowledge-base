```python
from datetime import datetime
import pytz

some_date = '2022.02.02 05:52:07 +04:00 UTC'

# 1.  Разбираем строку, чтобы получить datetime object с информацией о часовом поясе.
#    Обратите внимание: '%z' корректно обрабатывает смещение часового пояса.  Однако, 'UTC' часть необходимо отбросить.
date_format = '%Y.%m.%d %H:%M:%S %z UTC'

# Убираем " UTC" из строки, чтобы '%z' мог правильно ее разобрать
some_date_no_utc = some_date.replace(" UTC", "")

datetime_object = datetime.strptime(some_date_no_utc, date_format)


# 2.  Преобразуем datetime object в UTC.  Важно, чтобы datetime object знал о часовом поясе.
datetime_utc = datetime_object.astimezone(pytz.utc)

# 3. (Опционально) Выводим результат в нужном формате.
print(datetime_utc)
print(datetime_utc.strftime('%Y-%m-%d %H:%M:%S UTC'))
```
Объект datetime.datetime в Python может хранить следующую информацию:

•   Дата:
    •   Год (year): Целое число, представляющее год.
    •   Месяц (month): Целое число от 1 до 12, представляющее месяц.
    •   День (day): Целое число от 1 до 31, представляющее день месяца.

•   Время:
    •   Час (hour): Целое число от 0 до 23, представляющее час.
    •   Минута (minute): Целое число от 0 до 59, представляющее минуту.
    •   Секунда (second): Целое число от 0 до 59, представляющее секунду.
    •   Микросекунда (microsecond): Целое число от 0 до 999999, представляющее микросекунду.

•   Информация о часовом поясе (tzinfo):
    •   tzinfo: Объект, предоставляющий информацию о часовом поясе, связанном с объектом datetime. Это может быть None (если объект "naive", т.е. не имеет информации о часовом поясе) или экземпляр класса, производного от datetime.tzinfo (если объект "aware", т.е. знает о часовом поясе). Важно отметить, что часовой пояс не хранится в виде строки (например, "UTC" или "Europe/Moscow"), а в виде объекта tzinfo, который определяет правила смещения и переходов на летнее время.

Ключевые моменты:

•   Naive vs. Aware: Объекты datetime могут быть "naive" (не знают о часовом поясе) или "aware" (знают о часовом поясе).  Работать с aware-объектами гораздо безопаснее и рекомендуется, особенно если ваше приложение работает с датами и временем из разных часовых поясов.
•   Точность: datetime позволяет хранить дату и время с точностью до микросекунды.
•   Неизменяемость: Объекты datetime являются неизменяемыми.  После создания объекта его значения нельзя изменить.  Вместо этого создаются новые объекты.
•   Диапазон дат: Диапазон дат, которые можно представить с помощью datetime, обычно ограничен диапазоном, поддерживаемым операционной системой и базовой реализацией time.time().  Обычно это примерно от 1 января 0001 года до 31 декабря 9999 года.

Примеры:
```python
import datetime
import pytz

# Naive datetime object (не знает о часовом поясе)
naive_datetime = datetime.datetime(2023, 10, 27, 10, 30, 0)
print(naive_datetime)  # Output: 2023-10-27 10:30:00

# Aware datetime object (знает о часовом поясе)
utc_timezone = pytz.utc
aware_datetime = datetime.datetime(2023, 10, 27, 10, 30, 0, tzinfo=utc_timezone)
print(aware_datetime) # Output: 2023-10-27 10:30:00+00:00
```
