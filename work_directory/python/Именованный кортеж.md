Уточнения и дополнения:

1.  Не совсем словарь: Хоть ты и говоришь, что именованный кортеж это неизменяемый словарь - это не совсем так. Именованный кортеж это, скорее, класс, который автоматически создает структуру данных, похожую на кортеж, но с добавлением именованных полей. Это делает код более читаемым и понятным.

2.  Аргументы namedtuple:
    •   Первый аргумент namedtuple - это имя класса, который будет создан.  Это не просто строка, а имя, которое будет использоваться для создания новых экземпляров.
    •   Второй аргумент задает имена полей.  Это может быть строка с именами полей, разделенными пробелами (как в твоем примере), или список/кортеж строк. Например:
        
        Fruit = namedtuple("Fruit", ["name", "color"])  # Список
        Fruit = namedtuple("Fruit", ("name", "color"))  # Кортеж
        
        Разница лишь в стиле.

3.  Пример использования и преимущества:
```python
from collections import namedtuple

    # Создаем именованный кортеж "Point" с полями "x" и "y"
    Point = namedtuple("Point", ["x", "y"])

    # Создаем экземпляр именованного кортежа
    p = Point(x=10, y=20)

    # Доступ к полям по имени
    print(p.x)  # Вывод: 10
    print(p.y)  # Вывод: 20

    # Доступ к полям по индексу (как в обычном кортеже)
    print(p[0])  # Вывод: 10
    print(p[1])  # Вывод: 20

    # Именованные кортежи неизменяемы
    # p.x = 30  #  Вызовет ошибку AttributeError: can't set attribute

    # Преобразование в словарь (если нужно)
    print(p._asdict())  # Вывод: OrderedDict([('x', 10), ('y', 20)])
```
    
    •   Читаемость:  p.x гораздо понятнее, чем p[0], особенно если у кортежа много полей.
    •   Самодокументируемость:  Имена полей сразу показывают, что хранит каждый элемент.
    •   Безопасность:  Использование имен полей помогает избежать ошибок, связанных с неправильным порядком элементов.

4.  Когда использовать именованные кортежи?

    •   Когда нужна простая структура данных для хранения информации.
    •   Когда важна неизменяемость данных.
    •   Когда требуется легко читаемый и понятный код.
    •   Когда нужна небольшая экономия памяти по сравнению со словарями (особенно при создании большого количества экземпляров).
    •   Для возврата нескольких значений из функции (вместо простого кортежа).

5.  Методы и атрибуты:  У именованных кортежей есть несколько полезных встроенных методов и атрибутов:

    •  \_fields:  Возвращает кортеж строк, содержащих имена полей.
    •  \_make(iterable):  Создает новый экземпляр из итерируемого объекта (например, списка или кортежа).
    •   \_asdict():  Возвращает экземпляр OrderedDict, отображающий имена полей на их соответствующие значения.
