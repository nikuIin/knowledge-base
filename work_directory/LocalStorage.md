[[Сохраняем данные на фронтедне (frontend) Хранилища данных на фронтенде]]

Что важно знать о `localStorage`? 
Первое, в нем можно хранить только простые данные, а именно:
* `nubmer`
* `String`
* `bool`

Если ты туда хочешь засунуть какой-нибудь свой сложжннный объект, то использую `JSON.stringify(value)` для перевода в `String` и `JSON.parse(value)` для обратного перевода в `json/object`.

Вот пример:

```tsx
const user = {
  name: "Alex",
  age: 25
};

// Сохранение объекта в localStorage
localStorage.setItem('user', JSON.stringify(user));

// Извлечение объекта из localStorage
const retrievedUser = JSON.parse(localStorage.getItem('user'));
if (!retrievUser) return null // обязательно проверяй на null перед парсингом
console.log(retrievedUser.name); // "Alex"
```

Еще важно понимать, что мы не можем хранить большое количество данных. Разные браузера ограничивают ресурс хранения (ответ нейросетки):
* Safari **5 MB**
* Остальные *10 MB*
=> мы можем хранить только **5 МБ** данных. 

Какие поля мы можем хранить в локальном хранилище, даже лучше сказать так, какие проблемы может взять на себя локальное хранилище?

1. Настройки пользователя:
	Если наш сайт дает возможность настроить свой интерфейс, предпочтения и т.д. т.п., то будет хорошей идеей хранить эти данные в `localstorage`. *Например тему сайта, темная или светлая*
2. Какие-либо **НЕЧУВСТВИТЕЛЬНЫЕ ДАННЫЕ**. Я, например, хочу хранить игровой рейтинг в локальном хранилище, дабы не нагружать сервер ненужными запросами. Поставить `TTL` в 1 день и сверять статистику раз в день. Либо же переместить эти данные в [[session storage]].
3. Методы `localStorage` не работают в `async/await`, они все *синхронные*.
