[[Версионность git (коммиты)]]
Ветки в `git` создаются командой 

```bash
git branch my_branch
```

Нельзя создавать ветки с пробелами и с одинаковыми названиями. 

Чтобы изменить ветку в `git` нужно воспользоваться командой:

```bash
git switch my_branch
```

(До версии 2.23.0 использовалась `git checkout my_branch`, сейчас предпочтительнее `git switch my_branch`)

### Создаем и сразу переключаемся на ветку

```bash
git switch -c my_branch
```


# Что такое ветка?

`Ветка` – это ссылка на коммит. 
Если мы помним, то коммит (не первый) имеет ссылку на предыдущий коммит, его родителя, это связь однонаправлена, а также `hash`, по которому можно текущий коммит идентифицировать. Так вот, `branch` как раз — это просто ссылка на последний коммит. Когда в ветке добавляется новый коммит, она начинает ссылаться на него.

# Когда нам вообще использовать ветки?

Представьте такую ситуацию, вы работает над задачей А, вы ее еще не завершили, но к вам приходят и говорят, что нужно сделать быстрое изменение в задаче Б. Что происходит? Вы оставляете задачу А, делаете изменение (в текущей ветке) в Б, коммитете их и... в ваш коммит попадают недоделанные задачи из А. Это плохо. Коммит должен содержать адекватное законченное действие. Поэтому вам изначально нужно было сделать вот что:

1. Отслоиться от общей ветки на ветку задачи А
2. При получении задачи Б вернуться в общую ветку, сделать там изменения и закоммитить их
3. Вернуться на работу в ветку А

Какие ветки вообще есть?
---

Первая ветка **интеграционная** — это основная ветка проекта, где содержится его актуальная РАБОЧАЯ версия. Зачастую это дефолтная ветка `main/master`.

Все остальные ветки — **рабочие** или **функциональные**, их еще называют **топиками** или **темами**.
