[[git pull. Обновляем локальный репозиторий]]
[[Удаленная работа с репозиторием git]]

Как мы говорили в карточке о `git pull`, `git fetch` является ее частью.

Команда `git fetch` подгрузит **ВСЕ** ветки удаленного репозитория в локальный. Они все буду с префиксом *origin/* (стандартное название удаленной сети).

Эта команда только установит последние изменения для веток *origin*, ваши же ветки останутся в том же положении. Это очень удобная команда, ведь она позволяет вам решать, что делать дальше. Вы можете посмотреть изменения и решить, что делать дальше. 

Ветки с удаленного репозитория отличаются от ваших локальных. Они называются ветками слежения, помогают понять, в каком статусе сейчас репозиторий. Команда `git fetch` обновляет их, она может:
* создать ветки слежения (если они появились в результате вливания изменений другими пользователями)
* переместить их указатель по коммитами
* удалить с локального репозитория ветку слежения, если она удалилась в удалённом (если используется команда `git fetch --prune` или `git fetch -p`)

Чтобы скопировать ветку слежения, например *origin/cool-branch*, достаточно просто переместиться на нее без указания алиаса удаленного репозитория, то есть *origin*: `git switch cool-branch`

Чтобы вывести все ветки и соотношение веток с их ветками слежения:

```bash
git branch -vv --all
```

Чтобы посмотреть граф проекта полный:

```bash
git log --graph --oneline --all
```
