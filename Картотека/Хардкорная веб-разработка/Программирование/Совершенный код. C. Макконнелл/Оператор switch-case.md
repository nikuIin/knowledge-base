1. Оператор switch-case стоит использовать для проверок, которые можно разбить по простому на варианты, если проверка очень тяжелая, то лучше использовать if-elif-else.
2. Проверку располагайте от наиболее частой ситуации к менее частой. Если все варианты по вероятности примерно равны, то располагайте их в алфавитном порядке. 
3. Используйте оператор default для обработки ошибок, а ситуации закидывайте только в case
4. Не пишите большие коды в case, вместо этого оберните их в методы
5. Не используйте и case и if-elif-else рядом
6. Обработайте все ситуации в case, даже если код будет пустовать. 
7. Отмечайте вариант, который является верным комментариями
